# appexit.jp DNS設定手順

## 1. DNSレコードの設定

お名前.comまたは利用中のDNSプロバイダーの管理画面で以下を設定してください:

### Aレコードの追加

| ホスト名 | タイプ | 値 | TTL |
|---------|-------|-----|-----|
| @ (またはappexit.jp) | A | 54.250.252.70 | 3600 |
| www | A | 54.250.252.70 | 3600 |

### 設定例(お名前.comの場合)

1. お名前.comにログイン
2. 「DNS設定/転送設定」をクリック
3. 対象ドメイン「appexit.jp」を選択
4. 「次へ」をクリック
5. 「DNSレコード設定を利用する」の「設定する」をクリック
6. 以下のレコードを追加:
   - **TYPE**: A
   - **ホスト名**: (空白) ※ルートドメイン用
   - **VALUE**: 54.250.252.70
   - **TTL**: 3600
7. もう1つ追加:
   - **TYPE**: A
   - **ホスト名**: www
   - **VALUE**: 54.250.252.70
   - **TTL**: 3600
8. 「追加」ボタンをクリック

## 2. DNS反映の確認

設定後、以下のコマンドでDNSが反映されたか確認してください(最大48時間かかる場合がありますが、通常は数分〜1時間程度):

```bash
# ルートドメインの確認
nslookup appexit.jp

# wwwサブドメインの確認
nslookup www.appexit.jp
```

どちらも `54.250.252.70` が返ってくればOKです。

## 3. 次のステップ

DNSが反映されたら、以下のデプロイスクリプトを実行してください:
```bash
cd /path/to/appexit
sudo bash deploy/setup-nginx-ssl.sh
```

## トラブルシューティング

### DNS反映が遅い場合
- TTLを短く設定(300秒など)すると反映が早くなります
- キャッシュクリアを試してください: `sudo systemd-resolve --flush-caches` (Linux)

### nslookupで確認できない場合
- 別のDNSサーバーで確認: `nslookup appexit.jp 8.8.8.8`
- オンラインツールで確認: https://www.whatsmydns.net/
